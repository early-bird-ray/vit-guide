왜 ViT가 나왔을까?

AI가 이미지를 보는 방식이 진화하는 과정을 알아봅니다.

기존 방식: CNN (합성곱 신경망)

CNN은 작은 필터(커널)를 이미지 위에 슬라이딩하며 특징을 추출합니다.

마치 돋보기로 이미지를 부분부분 훑어보는 방식입니다.

✓ 작은 데이터에서도 잘 작동
✓ 위치 불변성 내장 (귀납적 편향)
✗ 멀리 떨어진 픽셀 관계 파악 어려움
✗ 모델 크기 키울수록 효율 저하
💡 핵심 아이디어의 시작
2017년 Google이 발표한 Transformer는 자연어처리(NLP)에서 혁명을 일으켰습니다.
문장의 모든 단어가 서로의 관계를 직접 학습하는 Self-Attention 덕분에,
"멀리 떨어진 단어들의 관계"를 완벽히 파악할 수 있게 되었습니다.

그렇다면 "이미지도 문장처럼 처리하면 어떨까?" — 이것이 ViT의 출발점입니다.

AI 비전 모델의 진화

1989 ~ 2012
CNN 시대
LeNet, AlexNet 등 합성곱 신경망이 이미지 인식을 지배. 필터로 지역 특징을 추출하는 방식.
2017
Transformer 등장 (NLP 혁명)
Google "Attention is All You Need" 발표. Self-Attention으로 NLP 모든 작업에서 SOTA 달성.
2019~2020
CNN + Attention 혼합 시도
CNN에 Attention을 더하는 방식들 등장. 하지만 CNN 구조 자체를 버리진 못함.
2020년 10월
ViT 발표 (Google Brain)
CNN을 완전히 제거한 순수 Transformer 비전 모델 최초 등장. 대규모 데이터에서 CNN 능가.
2021~현재
ViT 파생 모델 폭발적 증가
DeiT, Swin Transformer, BEiT, MAE, DINO 등 수많은 파생 모델이 비전 AI 판도를 바꾸는 중.
원리

ViT는 어떻게 작동할까?

이미지를 문장처럼 처리하는 5단계를 하나씩 살펴봅니다.

1
이미지를 패치(조각)로 자른다
224×224 이미지를 16×16 크기의 작은 조각으로 자릅니다.
총 (224÷16)² = 196개의 패치가 만들어집니다.
이 패치들이 NLP의 "단어 토큰"과 같은 역할을 합니다.
2
각 패치를 숫자 벡터로 변환한다 (Embedding)
각 16×16 패치(256개 픽셀 × 3채널 = 768개 숫자)를 선형 변환하여 D차원 벡터로 만듭니다.
또한 맨 앞에 [CLS] 토큰을 추가합니다 — 이게 최종 분류에 사용됩니다.
각 벡터에 위치 정보(Position Embedding)를 더해줍니다 (어느 위치의 패치인지 알아야 하므로).
3
Self-Attention: 모든 패치가 서로를 본다
ViT의 핵심입니다. 각 패치가 다른 모든 패치와의 관계를 동시에 계산합니다.
예를 들어 고양이 얼굴 패치는 꼬리 패치, 귀 패치와 높은 관계를 가질 수 있습니다.
CNN처럼 "주변만 보는 것"이 아니라 이미지 전체를 한번에 파악합니다.
4
Transformer Encoder를 여러 번 반복한다
Self-Attention + MLP 레이어를 12번(ViT-Base 기준) 반복합니다.
반복할수록 패치들은 점점 더 풍부한 맥락 정보를 얻습니다.
예를 들어 초반엔 "이 패치는 노란색"이었다면, 후반엔 "이 패치는 고양이 귀의 일부"가 됩니다.
5
[CLS] 토큰으로 최종 분류한다
12번의 Transformer를 통과한 후, [CLS] 토큰의 벡터를 분류 레이어에 넣습니다.
[CLS]는 학습을 통해 이미지 전체의 의미를 압축한 "요약 벡터"가 됩니다.
이 벡터가 선형 레이어를 거치면 "이 이미지는 고양이일 확률 92%"와 같은 결과가 나옵니다.

활용

ViT는 어디에 쓰일까?

이미지 분류를 넘어 다양한 분야로 확장되고 있습니다.

🔍

이미지 분류

ImageNet 1000개 클래스 분류. 의료 이미지 진단, 제품 불량 감지 등. 가장 기본적인 활용.

🎯

객체 탐지 (Detection)

DETR 등 Transformer 기반 탐지 모델의 백본으로 사용. "어디에 무엇이 있는지" 파악.

🎨

이미지 생성 (Diffusion)

Stable Diffusion, DALL-E 등 최신 생성 AI의 핵심 구성 요소로 Transformer 구조 활용.

🏥

의료 영상 분석

CT, MRI, 병리 슬라이드 분석. 전역 attention으로 미세한 이상 패턴 감지.

🤖

멀티모달 AI

CLIP, LLaVA 등 이미지+텍스트 이해 모델. "이 그림을 설명해줘" 같은 작업 가능.

🗺️

영상 분할 (Segmentation)

픽셀 단위 분류. 자율주행 도로 인식, 위성 이미지 지형 분석 등.

🎬

영상(Video) 이해

ViViT 등 시간 축을 추가한 변형. 행동 인식, 영상 분류 등 동영상 분석.

🔬

과학 데이터 분석

천문학 이미지, 물질 구조 분석, 기후 패턴 인식 등 과학 분야로 확장 중.

주요 파생 모델

DeiT
Facebook AI. 지식 증류로 적은 데이터에서도 ViT 학습 가능하게 개선.
Swin Transformer
Microsoft. 계층적 구조 + 이동 윈도우 attention으로 고해상도 효율화.
MAE
Facebook AI. 75% 패치를 마스킹하고 복원하는 자기지도 사전학습.
DINO
Facebook AI. 레이블 없이도 객체를 자동 분할하는 능력 습득.

HuggingFace 라이브러리로 ViT를 바로 실행해봅니다.

설치

이미지 분류 (추론)
from transformers import ViTImageProcessor, ViTForImageClassification
from PIL import Image
import requests

# 이미지 불러오기
url = "http://images.cocodataset.org/val2017/000000039769.jpg"
image = Image.open(requests.get(url, stream=True).raw)

# 모델 로드 (최초 실행 시 자동 다운로드)
processor = ViTImageProcessor.from_pretrained("google/vit-base-patch16-224")
model = ViTForImageClassification.from_pretrained("google/vit-base-patch16-224")

# 추론
inputs = processor(images=image, return_tensors="pt")
outputs = model(**inputs)
logits = outputs.logits

# 결과 출력
predicted_class = logits.argmax(-1).item()
print(f"예측: {model.config.id2label[predicted_class]}")
# 출력 예: 예측: Egyptian cat
내 데이터로 Fine-tuning
from transformers import ViTForImageClassification, TrainingArguments, Trainer

# 사전학습 모델 로드 (내 클래스 수로 교체)
model = ViTForImageClassification.from_pretrained(
    "google/vit-base-patch16-224-in21k",
    num_labels=10,                    # 내 데이터 클래스 수
    ignore_mismatched_sizes=True      # 분류 헤드 크기 변경
)

# 학습 설정
training_args = TrainingArguments(
    output_dir="./my-vit-model",
    num_train_epochs=5,
    per_device_train_batch_size=32,
    learning_rate=2e-5,
    evaluation_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True,
)
